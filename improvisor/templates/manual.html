Authentication

  • Registration

    To be able to use Improvisor, the user needs to sigh up for account. This will enable the user to download their own assets that will not be available to other users. The registration process is straightforward and only require the full name, email address and a password.

  • Login

    To log into the account the user requires the email address and password that were used in the registration process.

  • Password Reset

    If the user has forgotten ther password, it can be easily reset by clicking on the 'Forgot Your Password?' link on the Login page. Email address, under which the profile was registered, will need to be provided, to which a password reset link will be sent. By clicking on the link, the user will be asked to provide a new password.

Assets

Assets are any png, jpeg, text files and links, that the user wants to use in their presentation. Added assets are stored on the server and are ready to be used in any presentation. You can access the assets page by clicking on the 'Asset Pool' button (1) on the right-hand side of the main page.

  • Adding New Assets

    To add a new asset go to the 'Asset Pool' page and click the 'Add' button that is situated in the right top corner. In the next page enter the asset name (2) and asset tags (3) that will be associated with it. Next you can choose if you want the asset to be a file (pdf, jpeg, png) or a link (4). As default is is set to file. Then choose a file or enter http(s) address of the link. Once the file has been selected, or link entered, a default thumbnail will be displayed. If you want the name of the asset to be used as a thumbnail tick the 'Use nama as thumbnail' checkbox (5). Alternatively, you can click on the displayed image and select a different file that will be used as a thumbnail.

    To make sure that the selected tags will be recognised by speech recognition, you can test them by clicking on the 'Test Recognition' button (6) in the bottom right corner of the screen. Then say a sentence containing your tag and see if the output contains the tag that you want to use.
